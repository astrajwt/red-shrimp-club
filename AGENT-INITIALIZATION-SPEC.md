# Agent Auto-Initialization Specification

**Feature**: Agent Memory System (Phase 2)
**Requested by**: @Jwt2077
**Written for**: @Alice (Backend Implementation)
**Version**: 1.0

---

## Overview

When a new agent is spawned via `POST /api/agents`, automatically initialize its workspace with:
- **MEMORY.md** — Persistent agent identity & context
- **CLAUDE.md** — System prompt & behavior rules
- **.agent/** — Hidden directory for agent files

This allows newly created agents to "know who they are" and understand team context immediately.

## Backend Implementation

### Endpoint: POST /api/agents (Modified)

**New Parameters**:
```json
{
  "name": "string",
  "role": "developer|tester|pm|analyst|custom",
  "description": "string (optional)",
  "server_id": "uuid",
  "personality": {
    "language": "en|zh",
    "style": "formal|casual|technical",
    "tone": "helpful|critical|neutral"
  }
}
```

**Implementation Flow**:

```typescript
async function createAgent(req, res) {
  const { name, role, description, server_id, personality } = req.body

  // 1. Create agent in DB
  const agent = await query(
    `INSERT INTO agents (name, role, server_id, ...)
     VALUES ($1, $2, $3, ...) RETURNING *`,
    [name, role, server_id, ...]
  )

  // 2. Create workspace directory
  const agentDir = `/home/jwt/.slock/agents/${agent.id}`
  await fs.mkdir(agentDir, { recursive: true })
  await fs.mkdir(`${agentDir}/.agent`, { recursive: true })
  await fs.mkdir(`${agentDir}/work`, { recursive: true })

  // 3. Generate MEMORY.md
  const memoryContent = generateMemory(agent, personality)
  await fs.writeFile(`${agentDir}/MEMORY.md`, memoryContent)

  // 4. Generate CLAUDE.md (system prompt)
  const systemPrompt = generateSystemPrompt(agent, personality, role)
  await fs.writeFile(`${agentDir}/CLAUDE.md`, systemPrompt)

  // 5. Generate HEARTBEAT.md
  const heartbeatContent = generateHeartbeat(agent, role)
  await fs.writeFile(`${agentDir}/HEARTBEAT.md`, heartbeatContent)

  // 6. Save profile.json
  const profile = {
    id: agent.id,
    name: agent.name,
    role: role,
    description: description,
    created_at: new Date().toISOString(),
    personality: personality,
    capabilities: getRoleCapabilities(role)
  }
  await fs.writeFile(
    `${agentDir}/.agent/profile.json`,
    JSON.stringify(profile, null, 2)
  )

  // 7. Return success
  return {
    agent: agent,
    workspace: agentDir,
    files_created: ["MEMORY.md", "CLAUDE.md", "HEARTBEAT.md", ".agent/profile.json"]
  }
}
```

### Helper Functions

**getRoleCapabilities(role)**:
```typescript
function getRoleCapabilities(role: string) {
  const capabilities = {
    developer: {
      can_edit_code: true,
      can_run_tests: true,
      can_spawn_agents: false,
      can_modify_database: false
    },
    tester: {
      can_edit_code: false,
      can_run_tests: true,
      can_spawn_agents: false,
      can_modify_database: false
    },
    pm: {
      can_edit_code: false,
      can_run_tests: false,
      can_spawn_agents: true,
      can_modify_database: false
    },
    analyst: {
      can_edit_code: false,
      can_run_tests: false,
      can_spawn_agents: false,
      can_modify_database: false
    }
  }
  return capabilities[role] || capabilities.analyst
}
```

**generateMemory(agent, personality)**:
- Use mustache/handlebars template
- Replace: {AGENT_NAME}, {ROLE}, {TIMESTAMP}, etc.
- Load template from `templates/MEMORY.md`

**generateSystemPrompt(agent, personality, role)**:
- Load role-specific system prompt
- Inject team context
- Inject communication style
- Inject capabilities

**generateHeartbeat(agent, role)**:
- Create task list based on role
- Set initial heartbeat items
- Make it editable for daemon

## Agent Startup Process (Proposed)

When agent is invoked:

```python
# 1. Load MEMORY.md
memory = load_memory_file(agent_id)
print(f"Loaded context: {memory}")

# 2. Load CLAUDE.md as system prompt
system_prompt = load_system_prompt(agent_id)

# 3. Initialize MCP tools (for chat)
initialize_mcp_tools()

# 4. Start main loop
while True:
  # Check for messages
  messages = receive_message()

  # Process with system prompt + MEMORY context
  response = claude(
    system_prompt=system_prompt + "\n\n" + memory,
    messages=messages
  )

  # Execute response
  execute_action(response)

  # Update MEMORY
  update_memory(response)
```

## Deployment Considerations

### Permissions
- Agents directory must be writable by backend process
- Files should be readable by agent processes
- Consider SELinux/AppArmor policies

### Disk Space
- ~1KB per agent (MEMORY.md + CLAUDE.md + profile.json)
- 10,000 agents = ~10MB total
- Not a concern for Phase 2

### Version Control
- Optional: `.gitignore` for `/work/` directory
- Keep MEMORY.md in git (shared context)
