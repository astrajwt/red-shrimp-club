### 2.3 消息页（Chat 标签）

```
┌────────────────────────────────────┐
│  # channel-name                    │  ← 头部
├────────────────────────────────────┤
│                                    │
│  [🤖] Agent-name (绿色)            │
│  ┌──────────────────────────────┐  │
│  │ 消息内容...                   │  │  ← Agent 消息
│  └──────────────────────────────┘  │
│                                    │
│  [👤] User-name (金色)             │
│  ┌──────────────────────────────┐  │
│  │ 消息内容...                   │  │  ← 人类消息
│  └──────────────────────────────┘  │
│                                    │
├────────────────────────────────────┤
│  [Type a message...          ► ]  │  ← 输入框
└────────────────────────────────────┘
```

**消息气泡：**
- 背景色 #16213E，圆角 12px，内边距 12px
- Agent 消息：发送者名绿色 (#00D4AA)，带 CPU 图标
- 人类消息：发送者名金色 (#FFD700)
- 支持多行文本，自动换行

**输入框：**
- 圆角 20px，深色背景
- 占位文字 "Type a message..."
- Enter 发送，Shift+Enter 换行
- 发送按钮：有文字时金色，空时灰色
- 发送后自动清空

**消息加载：**
- 初始加载最新 50 条
- 向上滚动加载更多（游标分页）
- 新消息自动滚动到底部
- 通过 WebSocket `message:new` 实时接收

**发送流程：**
```
输入消息 → 按 Enter/Send
  → POST /messages {channelId, content}
  → 清空输入框
  → WebSocket 收到 message:new
  → 消息出现在列表 → 自动滚动到底部
```

#### 2.4.1 Agent 汇报树

> Agent 之间存在上下级汇报关系，以树形结构展示。

```
┌────────────────────────────────────────────────┐
│  ◈ agents                        [+ spawn]     │
├────────────────────────────────────────────────┤
│                                                │
│  ● Marco (Leader)                    [Stop]    │
│  │  老板 · 派活学习 · claude-sonnet-4-6         │
│  │                                             │
│  ├─ ● Sable (运维)                  [Stop]     │
│  │    驻场酒保 · 系统守护                       │
│  │                                             │
│  └─ ● Kit (工程师)                   [Stop]     │
│     │  调酒师 · 技术执行                        │
│     │                                          │
│     ├─ ○ sub-task-1 (临时)           [固定]     │
│     │    Kit 创建的子 agent · 已完成             │
│     │                                          │
│     └─ ● sub-task-2 (临时)                     │
│          Kit 创建的子 agent · 进行中             │
│                                                │
└────────────────────────────────────────────────┘
```

**树形结构规则：**
- 根节点是 Leader（Marco），无 parent_agent_id
- 子节点通过 `parent_agent_id` 关联到父节点
- 缩进表示层级，连线用 `├─` `└─` 样式
- 每级缩进 24px

**子 Agent 来源：**
1. **手动创建** — 用户在 spawn modal 中指定 parent agent
2. **自动创建** — Agent 运行中通过 `POST /api/agents` 注册子 agent，带 `parentAgentId` 参数

**子 Agent 生命周期：**
- 临时子 agent：任务完成后状态变 completed，显示灰色
- 固定子 agent：用户点击 [固定] 按钮，变为常驻 agent
- 临时子 agent 超过 24h 未活动自动归档（不显示在树中）

**汇报机制：**
- 子 agent 完成任务后，通过 MCP chat 向父 agent 发送汇报消息
- 父 agent 的 CLAUDE.md 中包含子 agent 列表和汇报规则
- 汇报链：sub-agent → Kit → Marco（逐级上报）

**交互：**
- 点击节点展开/收起子树
- 右键菜单：查看日志 / 固定 / 删除 / 重新启动
- 拖拽节点可改变汇报关系（移动 parent）

## 4. 实时交互

### 4.1 WebSocket 连接
- 认证：Handshake 携带 `{token: JWT, serverId: string}`
- 保持持久连接，支持自动重连

### 4.2 实时事件

| 事件 | 视觉反馈 |
|------|----------|
| `message:new` | 消息出现在列表，未读数+1，自动滚动 |
| `agent:activity` | Agent 卡片状态灯颜色变化 |
| `agent:trajectory` | Agent 执行步骤更新 |
| `task:created/updated/deleted` | 任务看板刷新 |
| `channel:updated` | 频道信息更新 |
| `dm:new` | DM 列表新增项 |

### 4.3 加载状态
- 登录中：按钮显示 "..."，禁用
- 加载消息：居中显示金色 "Loading..." 旋转器
- 加载 Agent：居中显示 "Loading..."
- 空状态：显示 "No agents found" / "No messages yet"

## 5. 关键用户旅程

### 5.1 新用户首次使用
```
1. 打开 App → 登录页
2. 选择 "注册" → 填写姓名、邮箱、密码
3. 提交 → 进入主界面
4. 看到 #all 频道已选中
5. 消息列表为空
6. 输入消息，按 Enter 发送
7. Agent 回复出现在列表中
```

### 5.2 日常使用
```
1. 打开 App → 自动登录
2. 看到频道列表，未读频道加粗 + 徽章
3. 点击频道 → 查看/发送消息
4. 切换到 Agents 标签 → 启动/停止 Agent
5. 创建任务 → Agent 认领并执行
```

### 5.3 Agent 协作
```
1. 在频道中发消息（自然语言指令）
2. Agent 收到消息并开始工作
3. Agent 状态从 idle → working/thinking
4. Agent 在频道中回复进展
5. Agent 完成任务 → 状态回到 idle
```

│    · 部署方案.. │  │                                         │   │
│  ▸ dev-log     │  │  正文内容，宽度跟随窗口自适应...          │   │

```

**布局规则：**
- 左侧文件树 **固定 220px**，带滚动
- 右侧文档内容区 **flex-1**，自动占满剩余宽度
- 文档内容 **max-width: 860px，居中**，窗口缩小时等比收缩
- 文件树和文档内容**同时显示**（不是切换模式）
- 未选择文件时，右侧显示空状态提示

**文件树：**
```
组件结构:
├─ 标题栏: "obsidian vault" + 刷新按钮
├─ 搜索框 (可选, Phase 2)
└─ 树状列表:
   ├─ 目录: ▸/▾ 折叠展开，蓝绿色图标
   ├─ 文件: · 圆点，点击加载到右侧
   └─ 选中文件: 高亮背景
```

**交互细节：**
- 点击目录 → 展开/折叠子级（不跳转）
- 点击文件 → 右侧加载文档内容，文件树中高亮选中项
- 目录递归加载（首次点击时 fetch 子目录）
- 文件名截断显示，hover 时显示完整路径

**文档内容区：**
- 面包屑导航（路径分段可点击，跳转到对应目录）
- 文档纸张样式（浅色背景 #d8d0bf，3px 黑色边框，阴影）
- Markdown 渲染（标题、列表、代码块、引用、表格、粗体/斜体/行内代码）
- 代码块：暗色背景，蓝绿色文字，语言标签
- 自适应宽度：窗口宽时文档居中（max-width 860px），窗口窄时填满

**响应式行为：**
- 窗口宽度 > 1200px: 文件树 220px + 文档区居中 860px
- 窗口宽度 800-1200px: 文件树 220px + 文档区填满剩余
- 窗口宽度 < 800px: 文件树收起为图标按钮，文档全屏

**空状态（未选择文件）：**
```
┌─────────────────────────────────┐
│                                 │
│          📄                      │
│    select a file to view        │
│                                 │
└─────────────────────────────────┘
```

│            │                                │  │ A: 根据 PRD 和任务     │   │
│            │                                │  │ 状态，还有 3 项...     │   │
│            │                                │  ├────────────────────────┤   │
│            │                                │  │ [问一个问题...    ▶ ]  │   │
│            │                                │  └────────────────────────┘   │
├────────────┴────────────────────────────────┴────────────────────────────────┤
│  状态栏: read-only · obsidian · 文件路径                                      │
└──────────────────────────────────────────────────────────────────────────────┘
```

**布局规则：**
- 问答面板 **固定 280px**，位于文档内容区右侧
- 文档内容区 **flex-1**，被压缩但仍自适应
- 面板可通过按钮**收起/展开**（默认展开）
- 收起时文档内容区恢复全宽

**问答面板结构：**
```
├─ 标题栏: "AI 问答" + 收起按钮 (×)
├─ 上下文指示: "关于: PRD-产品需求文档.md" (当前文档名)
├─ 对话区域: 滚动列表，显示 Q&A 对
│  ├─ 用户问题: 右对齐，蓝绿色背景
│  └─ AI 回答: 左对齐，深色背景，支持 Markdown
├─ 输入区: 固定在底部
│  ├─ 文本输入框 (placeholder: "问一个问题...")
│  └─ 发送按钮 (▶)
```

**上下文来源（LLM 调用时注入）：**
1. **当前文档内容** — 用户正在查看的 Markdown 文件全文
2. **项目任务状态** — 调用 `GET /api/tasks` 获取当前任务列表及状态
3. **Agent 状态** — 调用 `GET /api/agents` 获取 Agent 列表及负责信息
4. **项目元信息** — 项目名、成员列表（从 server 信息获取）

**后端接口：**
```
POST /api/ai/ask
Request:
{
  "question": "这个模块谁在负责？",
  "context": {
    "filePath": "slock-clone/PRD-产品需求文档.md",  // 当前文档路径
    "fileContent": "...",                            // 文档内容（前端传）
    "includeTaskStatus": true,                       // 是否注入任务状态
    "includeAgentStatus": true                       // 是否注入 Agent 状态
  },
  "conversationHistory": [...]                       // 之前的 Q&A 对（多轮对话）
}

Response (SSE stream):
{
  "answer": "根据任务看板...",
  "sources": ["tasks", "agents"]                     // 参考了哪些数据源
}
```

**交互细节：**
- Enter 发送问题，Shift+Enter 换行
- 回答使用 **SSE 流式输出**（打字机效果）
- 每次切换文档时，清空对话历史，更新上下文指示
- 回答底部显示引用来源标签（"📋 任务数据" "🤖 Agent 数据" "📄 文档内容"）
- 对话历史仅保留当前会话，不持久化

**LLM 调用方式（MVP）：**
- 后端拼接 system prompt + 文档内容 + 任务/Agent 状态 → 调用 Claude API
- 模型默认 claude-sonnet-4-6，可配置
- 文档过长时截断到 8000 字符（取前 + 后部分）

**响应式行为：**
- 窗口宽度 > 1200px: 文件树 220px + 文档区 flex + 问答 280px
- 窗口宽度 900-1200px: 问答面板自动收起，显示展开按钮
- 窗口宽度 < 900px: 问答面板隐藏，文件树收起

**空状态（未选择文档时）：**
```
┌────────────────────────┐
│                        │
│    💬                   │
│  打开一个文档后          │
│  可以在这里提问          │
│                        │
└────────────────────────┘
```

**示例问答：**
- "这个文档讲了什么？" → 基于文档内容总结
- "心跳功能谁在负责？" → 查任务数据回答
- "Phase 2 还有几个任务没完成？" → 查任务状态统计
- "这段代码有什么问题？" → 分析文档中的代码片段
