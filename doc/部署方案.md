## 1. 服务器环境

### 1.1 硬件
- **阿里云 ECS** 2C8G（2 核 CPU, 8GB RAM）
- 系统盘 40GB SSD + 数据盘 100GB SSD
- Ubuntu 22.04 LTS

### 1.2 网络
- 公网 IP（弹性 EIP）
- 安全组规则：

| 端口 | 用途 | 来源 |
|------|------|------|
| 22 | SSH | 管理员 IP |
| 80 | HTTP → Nginx | 0.0.0.0/0 |
| 443 | HTTPS → Nginx | 0.0.0.0/0 |
| 5432 | PostgreSQL | 127.0.0.1 only |
| 3000 | API Server（内部） | 127.0.0.1 only |

### 3.2.1 局域网访问关键项

- 如果需要从同一局域网的其他设备访问 `http://<本机IP>:3001`，`HOST` 必须设置为 `0.0.0.0`
- 如果误写成 `127.0.0.1`，常见现象是：
  - 本机 `http://127.0.0.1:3001` 可访问
  - `http://<局域网IP>:3001` 返回 `ERR_EMPTY_RESPONSE`、`ERR_CONNECTION_REFUSED` 或无响应

推荐检查命令：

```bash
grep '^HOST=' backend-src/.env
ss -lntp | grep ':3001'
curl -I http://127.0.0.1:3001/
curl -I http://$(hostname -I | awk '{print $1}'):3001/
```

### 3.2.2 页面无法打开 / `ERR_EMPTY_RESPONSE` 排障

这次项目里实际踩到过两类不同根因，现象会很像“页面打不开”。

#### 情况 A：局域网地址打不开，但本机 `127.0.0.1:3001` 正常

根因通常是 backend 只监听在回环地址：

```bash
# 错误示例
HOST=127.0.0.1
```

排查方式：

```bash
grep '^HOST=' backend-src/.env backend-src/.env.dev
ss -ltnp | grep 3001
curl -I http://127.0.0.1:3001/
curl -I http://$(hostname -I | awk '{print $1}'):3001/
```

如果 `ss` 输出是 `127.0.0.1:3001`，说明局域网流量进不来。修复方式：

```bash
# backend-src/.env
HOST=0.0.0.0

# backend-src/.env.dev
HOST=0.0.0.0
```

然后重启 backend，再确认监听地址已经变成：

```bash
ss -ltnp | grep 3001
# 期望看到 0.0.0.0:3001
```

#### 情况 B：浏览器打开页面后白屏 / 黑屏 / “未发送任何数据”

根因可能不是 React 本身，而是前端静态资源请求被错误回退成了 `index.html`。

这次实际问题出在 `backend-src/src/index.ts` 的 Fastify Static 配置。若静态资源匹配失败，而 `setNotFoundHandler()` 又把所有非 `/api` 路径都回退到 `index.html`，浏览器请求 `/assets/index-*.js` 时就可能拿到一份 HTML，导致前端 bundle 根本跑不起来。

重点检查：

```bash
curl -I http://127.0.0.1:3001/assets/index-*.js
```

正确结果应为：

```text
content-type: application/javascript
```

如果返回的是：

```text
content-type: text/html
```

说明静态资源被 SPA fallback 吃掉了。

当前项目已知安全配置是：

```ts
await app.register(import('@fastify/static'), {
  root: frontendDir,
  prefix: '/',
  decorateReply: false,
})
```

不要再加会破坏资源命中的额外限制；否则很容易出现：
- 首页 HTML 能打开
- `/health` 是 `200`
- 但浏览器实际加载不到真正的 JS/CSS，最终表现为白屏、黑屏、`ERR_EMPTY_RESPONSE` 或“未发送任何数据”

## 4. systemd 服务配置

### 4.1 API Server

```ini
# /etc/systemd/system/redshrimp-api.service
[Unit]
Description=Red Shrimp Lab API Server
After=network.target postgresql.service

[Service]
Type=simple
User=jwt
ExecStart=/usr/bin/node /opt/redshrimp/api/dist/index.js
Restart=always
RestartSec=5
Environment=NODE_ENV=production
EnvironmentFile=/opt/redshrimp/.env
WorkingDirectory=/opt/redshrimp/api
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

### 4.2 Daemon

```ini
# /etc/systemd/system/redshrimp-daemon.service
[Unit]
Description=Red Shrimp Lab Agent Daemon
After=network.target postgresql.service redshrimp-api.service

[Service]
Type=simple
User=jwt
ExecStart=/usr/bin/node /opt/redshrimp/daemon/dist/index.js
Restart=always
RestartSec=5
Environment=NODE_ENV=production
EnvironmentFile=/opt/redshrimp/.env
WorkingDirectory=/opt/redshrimp
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

### 4.3 启用服务

```bash
sudo systemctl daemon-reload
sudo systemctl enable redshrimp-api redshrimp-daemon
sudo systemctl start redshrimp-api redshrimp-daemon

# 查看状态
sudo systemctl status redshrimp-api
sudo systemctl status redshrimp-daemon

# 查看日志
journalctl -u redshrimp-api -f
```

## 5. Nginx 反向代理

```nginx
# /etc/nginx/sites-available/redshrimp
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /opt/redshrimp/web/dist;
        try_files $uri $uri/ /index.html;
    }

    # API 代理
    location /api/ {
        proxy_pass http://127.0.0.1:3000/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # WebSocket 代理
    location /socket.io/ {
        proxy_pass http://127.0.0.1:3000/socket.io/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 文件上传大小限制
    client_max_body_size 50M;
}
```

```bash
# 启用配置
sudo ln -s /etc/nginx/sites-available/redshrimp /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx

# HTTPS（可选，有域名后执行）
sudo certbot --nginx -d your-domain.com
```

## 6.3 前端可访问性排障

### 症状 A：局域网地址返回 `ERR_EMPTY_RESPONSE`

先检查 backend 是否真的监听在局域网地址上：

```bash
ss -lntp | grep ':3001'
curl -I http://127.0.0.1:3001/
curl -I http://$(hostname -I | awk '{print $1}'):3001/
```

如果 `127.0.0.1` 正常、局域网 IP 异常，优先检查 `.env` 中是否误配：

```bash
HOST=127.0.0.1
```

应改为：

```bash
HOST=0.0.0.0
```

然后重启 backend。

### 症状 B：页面黑屏 / 浏览器说脚本不是有效 JS

这通常不是前端 bundle 本身坏了，而是 backend 的 SPA 回退逻辑把 `/assets/*.js` 也错误地回退成了 `index.html`。

快速验证：

```bash
curl -I http://127.0.0.1:3001/assets/<入口js文件名>.js
```

正确结果应包含：

```text
content-type: application/javascript
```

如果返回成 `text/html`，说明静态资源路由配置有误。修复原则：

- `/assets/*` 必须优先命中真实静态文件
- `/api/*`、`/internal/*`、`/uploads/*`、`/socket.io*`、`/daemon/*` 不能走 SPA 回退
- 只有普通前端路由才回退到 `index.html`

## 7. 部署脚本

### 7.1 一键部署 (deploy.sh)

```bash
#!/bin/bash
# /opt/redshrimp/scripts/deploy.sh
set -e

echo "=== Red Shrimp Lab Deploy ==="

cd /opt/redshrimp
git pull origin main

echo "--- Building API ---"
cd api && npm ci && npm run build

echo "--- Building Daemon ---"
cd ../daemon && npm ci && npm run build

echo "--- Building Web ---"
cd ../web && npm ci && npm run build

echo "--- Restarting services ---"
sudo systemctl restart redshrimp-api
sudo systemctl restart redshrimp-daemon

echo "--- Checking status ---"
sleep 2
sudo systemctl status redshrimp-api --no-pager
sudo systemctl status redshrimp-daemon --no-pager

echo "=== Deploy complete ==="
```

### 7.2 数据库备份 (backup.sh)

```bash
#!/bin/bash
# /opt/redshrimp/scripts/backup.sh
BACKUP_DIR="/var/slock/backups"
DATE=$(date +%Y%m%d_%H%M%S)

mkdir -p $BACKUP_DIR

# 数据库备份
pg_dump -U redshrimp redshrimp > "$BACKUP_DIR/db_$DATE.sql"

# 上传文件备份
tar czf "$BACKUP_DIR/uploads_$DATE.tar.gz" /var/slock/uploads/

# 保留最近 7 天
find $BACKUP_DIR -mtime +7 -delete

echo "Backup completed: $DATE"
```

### 7.3 定时备份（Cron）

```bash
# 每天凌晨 3 点备份
0 3 * * * /opt/redshrimp/scripts/backup.sh >> /var/log/redshrimp-backup.log 2>&1
```
