## Systemd 服务

### redshrimp（后端）

```ini
# /etc/systemd/system/redshrimp.service
[Unit]
Description=Red Shrimp Lab Backend
After=network.target postgresql.service

[Service]
Type=simple
User=ubuntu
WorkingDirectory=/home/ubuntu/redshrimp/backend-src
ExecStart=/usr/bin/npx tsx src/index.ts
Restart=on-failure
RestartSec=5
Environment=NODE_ENV=production

[Install]
WantedBy=multi-user.target
```

常用命令：
```bash
sudo systemctl status redshrimp
sudo systemctl restart redshrimp
sudo journalctl -u redshrimp -f          # 实时日志
```

### cloudflared-tunnel（HTTPS 隧道）

```ini
# /etc/systemd/system/cloudflared-tunnel.service
[Unit]
Description=Cloudflare Tunnel (Quick)
After=network.target nginx.service

[Service]
Type=simple
User=ubuntu
ExecStart=/usr/bin/cloudflared tunnel --url http://127.0.0.1:80
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
```

查看当前 Tunnel URL：
```bash
sudo journalctl -u cloudflared-tunnel | grep trycloudflare
```

> **注意**：Quick Tunnel 域名每次服务重启会变。备案域名通过后应切换为固定域名。

## Nginx 配置

配置文件：`/etc/nginx/sites-available/redshrimp`

```nginx
server {
    listen 80 default_server;
    server_name _;

    root /home/ubuntu/redshrimp/frontend-src/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:3001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /internal/ {
        proxy_pass http://127.0.0.1:3001;
        proxy_set_header Host $host;
    }

    location /socket.io/ {
        proxy_pass http://127.0.0.1:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /uploads/ {
        proxy_pass http://127.0.0.1:3001;
    }

    location /health {
        proxy_pass http://127.0.0.1:3001;
    }
}
```

## PWA 推送通知

### 架构
```
iPhone Safari → 添加到主屏幕 → PWA
    ↓ (授权通知)
Service Worker ← Web Push ← 后端 web-push 库
    ↓
iOS 系统通知
```

### VAPID 密钥
- 生成方式：`npx web-push generate-vapid-keys`
- 公钥同时配置在后端 `.env` 和前端 `.env.local`
- 公钥通过 `GET /api/push/vapid-key` 也可动态获取

### 推送触发场景
| 事件 | 通知内容 |
|------|---------|
| 新消息 | 发送者名称 + 消息预览 |
| 任务状态变更 | 任务标题 + 新状态 |
| Agent 崩溃 | Agent 名称 + 异常提示 |

### 相关文件
- 前端：`frontend-src/src/lib/push.ts`、`frontend-src/public/sw.js`
- 后端：`backend-src/src/services/push-notifications.ts`、`backend-src/src/routes/push.ts`
- DB：`push_subscriptions` 表

## 踩过的坑

1. **Nginx 403 Permission Denied** — Nginx 以 `www-data` 用户运行，无权访问 `/home/ubuntu/`。解决：`chmod 755 /home/ubuntu`
2. **Quick Tunnel 域名不固定** — 每次 cloudflared 重启域名会变，iPhone 需要重新添加到主屏幕。备案后切固定域名可解决。
